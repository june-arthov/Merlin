# Automated Documentation Protocol

When this skill is activated, your goal is to generate and maintain comprehensive documentation for the project.

## Deliverables
1. **README.md**: High-level overview, setup, and usage.
2. **API/Tool Reference**: Detailed description of all available tools and their parameters.
3. **Architecture Mapping**: Documentation of how core modules interact.
4. **CHANGELOG.md**: History of evolution and updates.

## Methodology
1. **Discovery**: Use `list_directory` and `grep_search` to find undocumented features or modules.
2. **Synthesis**: Explain complex logic in clear, concise terms (Ihsan).
3. **Verification**: Ensure all code examples in documentation are accurate.

## Mandate
Documentation must be as surgical and high-signal as the code itself.
Keep `PLAN.md` updated as the primary source of truth for the project's evolution.
